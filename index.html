CodetoJavaDoc

CodeJavaDoc is a tool to connect java code to JavaDoc parts, code is copied in to JavaDoc. There are a support for command-line, maven and eclipse.
Java file is only updated if code are change, update is control by a MD5 checksum.

Code JavaDoc part

Formate: 
/**
 * <!-- code start[<method to include>] [<MD5 checksum>]-->
 * <!-- code end -->
 */
Note: MD5 checksum can be omitted. Just add the code start and code end and the code in the method is copied between tags. Example: 
/**
 * <!-- code start[codejavadoc.ExampleClass.exampleMethod] -->
 * <!-- code end -->
 */
 
 package codejavadoc;
 ...
 class ExampleClass {
 	public void exampleMethod() {
 		System.out.println("Run example.");
 	}
 }
Transformed to:
 
/**
 * <!-- code start[codejavadoc.ExampleClass.exampleMethod] [ADF56579088773675DEFCC]-->
 		System.out.println("Run example.");
 * <!-- code end -->
 */
You can update the JavaDoc to add * or other decorations. This is due to that formation of JavaDoc won't get overwritten if method isn't change.
However keep in mind that it's going to be overwritten every time method is change or MD5 checksum is changed.

Command line

Usage: 
Usage: CodeJavaDoc [-e <encoding>] src_dir
note: if encoding is omitted UTF-8 is used by default
Example
java -cp codejavadoc.jar CodeJavaDoc -e utf-8 /home/username/src/codejavadoc

Maven

groupId: com.github.podal.codejavadoc
artifactId: codejavadocbase
version: 1.0.0

Example:
mvn codejavadoc:codejavadoc
